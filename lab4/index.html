Simple Task Outline: Changing a tire

  1. Gather tools/materials
    • Spare Tire
    • Tire iron
    • Jack
    • Tire Wedges
  2. Setup
    • Loosen Lugs
    • Wedge Tires
  3. Lift Car
    1. Locate solid area of frame
    2. Place jack under
    3. lift the car with the jack
  4. Tire Removal
    1. Remove lugs fully with tire iron
    2. Remove Tire
  5. Attach Spare
    1. Mount Spare
    2. Secure spare tire with lugs
  6. Finishing Up
    1. Lower car with jack
    2. Give lugs a final tightening
    3. Put away tools and materials

Creating Connect Four:

  1. Print the board before each turn
  2. Accept user input
  3. Validate user move (check if column full)
  4. Reprompt if invalid move
  5. Place tile at topmost unfilled spot in chosen column
  6. Check if Four are in a row
  7. Repeat until board filled or Four in a Row

Challenge

This lab challenged me to think about problems and tasks from a high level. Rather than just jumping in and thinking about the core of the task I was forced to think about the task from the root and build up.

Problems

When designing the PsuedoCode I struggled to contextualize how I would build it up. However, using the list I created to break each point into a sub heading for the psuedocode I was able to build each task one at a time and accomplish the task.

Reflection

This assignment was very helpful in redifining how I think about problems. Rather than jumping straight in, I had to take a step back and think about the task overall. This helped me design the other parts of the lab. Overall, I think this lab was very helpful not only in this context, but changing how I approach things in the future.
